Shell Sort

Definition: It is an in-place comparison sorting algorithm and an optimized version of the insertion sort.


How it works:
The core idea is to break the original list into several smaller sublists based on a specific "gap" sequence.
1.Choose a Gap Sequence 
2.Sort Sublists
3.Reduce the Gap
4.Final Pass


Characteristics:
1.Time Complexity
2.Space Complexity: It is an in-place sorting algorithm
3.Stability: It is not a stable sort(equal elements may not retain their original relative order).
4.Adaptability: It performs well


Time Complexity:
The time complexity of the Shell Sort algorithm is highly dependent on the chosen gap sequence and is not a single, fixed value.
  *Best Case: O(n log n), when the array is already nearly sorted (depends on gap sequence).
  *Average Case: Between O(n**1.25) and O(n**1.5), depending on the chosen gap sequence.
  *Worst Case: O(n**2), with simple gap sequences like n/2, n/4, …, 1.
Followed Key Details:
1.Dependency on Gap Sequence:
  *Common gap sequences (e.g., Knuth, Hibbard, Sedgewick) are designed to improve performance over the basic O(n**2) worst case.
2.Optimization of Insertion Sort:
  *By moving elements over large distances initially, 
   Shell Sort reduces the number of swaps needed in the final, small-gap passes, significantly improving practical performance.
