Introduction to Binary Tree

Definition: It is a tree data stucture in which each node has at most children(referred to the left and the right child).


Stucture: The top of the node is called the root, and the bottom nodes are called leaves.


Each node in a Binary Tree has three parts:
1.Data
2.Pointer to the left child
3.Pointer to the right child


The words in Binary Tree
1.Parent Node: A node that is the direct ancestor of a node.
2.Child Node: A node that is the direct descendant of another node.
3.Subtree of a node: A tree consisting of that node as root and all its descendants.
4.Path in a binary tree: A sequence of nodes connected by edges from one node to another.


Key Characterisitics
1.Node Structure: Each node contains a value and pointers to its left and right children.
2.Root: The top-most node of the tree.
3.Leaf: A node with no children.
4.Height: The number of edges on the longest path from the root to a leaf.


Types of Binary Trees
1.Full Binary Tree: Every node has either 0 or 2 children.
2.Complete Binary Tree: All levels are filled except possibly the last, which is filled from left to right.
3.Perfect Binary Tree: All internal nodes have two children and all leaf nodes are at the same level.
4.Skewed Binary Tree: A pathological tree where nodes only have left or only have right children.


The Uses of Various Trees 
Binary Search Trees (BST): Used for fast searching, sorting, and data retrieval (O(log n) average time complexity).
Decision Trees: Used in machine learning for classification and regression.
Expression Trees: Used by compilers to parse expressions.
Huffman Coding Trees: Used in data compression algorithms.


Common Traversal Methods
Inorder (Left, Root, Right): Produces sorted output for a BST.
Preorder (Root, Left, Right): Useful for creating a copy of the tree.
Postorder (Left, Right, Root): Useful for deleting the tree.
Level-Order: Visits nodes level-by-level.
